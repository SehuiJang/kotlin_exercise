# What chapter in 'Kotlin in Action'?
Chapter 1.1 A taste of Kotlin

# What did I look?
1. defining a data class
```
data class Person(val name: String, 
                    val age: Int? = null
                    )
```
2. nullable type: 'Int?', and default value for argument
```
val age: Int? = null
```
3. top-level function
Top-level function is a function declared directly inside a file, outside of any class, object, or interface.
```
fun main(args: Array<String>) {
    ...
}
```
4. declare and create a list / named argument
```
val persons = listOf(Person("Alice"),
                        Person("Bob", age = 29) // -> in this case, 'age = 29' is a named argument.
                        )
```
5. lambda expression and elvis operator(?:)
```
// Especially, this might be Training Lambda: if the last argument of a function is lambda expression, you can move the lambda outside parenthesis
// keyword 'it': when a lambda has only one parameter, you can omit the parameter declaration and refer to the argument using 'it'.

val oldest = persons.maxBy { it.age ?: 0 } 
// lambda expression is from '{' to '}'.
// 'it.age' is used instead of 'x -> x.age'
```
6. string template and string formatting
```
// string template
println("(string template)The oldest is: $oldest")

// string formatting
println("(string.format())The oldest is: %s".format(oldest))
```
7. autogenerated 'toString': when an object belonging to Person class is printed by println, its 'toString'(e.g., Person(name=Bob, age=29)) is autogenerated and then printed.
```
println("(string.format())The oldest is: %s".format(oldest))
// output: (string.format())The oldest is: Person(name=Bob, age=29)
```
