# How to call top-level functions from Java?
> Unless otherwise stated, all explanations in this document assume **Kotlin/JVM**.

## Create a sample `.kt` file
Well, I created a `.kt` file, following 'Kotlin in Action 2/e'.  
```kotlin
// join.kt
package strings

fun joinToString( /* ... */ ): String { /* ... */ }
```
When compiling this file, some classes are made because the compilation result is JVM bytecode.  
Importantly, JVM can only execute code in classes.
- On the JVM, a function has to be compiled into a method that belongs to a class.

If you just use Kotlin, that's all.  
Conversely, it is worth understanding how it will be compiled, if you want to call this function from Java.

## Equivalent Java code
There is a Java code, which would compile to the same class as `join.kt`.  
```java
/* Java */
package strings;

// JoinKt: corresponds to the filename `join.kt`
public class JoinKt {
    public static String joinToString(/* ... */) { /* ... */ }
}
```
- The name of the class generated by kotlinc corresponds to the filename `join.kt`.
  - The filename `join.kt` is capitalized &rarr; `Join` (Java's naming scheme) + `Kt` (suffix) &rarr; `JoinKt`
- Note that top-level functions in `.kt` file are compiled to `static` methods within the class.

When you want to call the function `joinToString` from Java, you can take the following way:
```java
/* Java */
import strings.JoinKt; // {package name in Kotlin}.{capitalized filename (Join) & Kotlin suffix (Kt)}

/* any code... */

JoinKt.joinToString(list, ", ", "", "");
```

## Changing the file class name
As discussed above, the kotlinc-generated class name (i.e., JVM class name) is determined from the filename with a suffix `Kt`.  
- Kotlin code file `join.kt` &rarr; kotlinc &rarr; class name `JoinKt`

Then, you can assign a JVM class name arbitrarily.
```kotlin
// join.kt
@file:JvmName("StringFunctions")    // a file-wide annotation: you can assign the JVM class name

package strings

fun joinToString(/* ... */): String { /* ... */ }
```

In this case, you can invoke this top-level function from Java:
```java
/* Java */
import strings.StringFunctions; // Not `JoinKt` but `StringFunctions` which is the assigned name.

StringFunctions.joinToString(list, ", ", "", "");
```

# How to call top-level properties from Java?
> Unless otherwise stated, all explanations in this document assume **Kotlin/JVM**.

## Create a sample `.kt` file
To explore how Java users access `val`, `var`, and `const val` in a `.kt` file, let me introduce an example:
```kotlin
// someFile.kt
package strings

var opCount = 0

val UNIX_LINE_SEPARATOR = "\n"

const val CONST_UNIX_LINE_SEPARATOR = "\n"
```

## Equivalent Java code
If you want to invoke properties in `.kt` file from Java, you should come up with the equivalent Java code:
```java
/* Java */
package strings;

public final class SomeFileKt {     // Kotlin filename `someFile.kt` -> Java class name SomeFileKt

    // Corresponding to `var` property `opCount` in Kotlin
    private static int opCount = 0;     // backing field

    public static final int getOpCount() {  // getter
        return opCount;
    }

    public static final void setOpCount(int value) {    // setter
        opCount = value;
    }

    // ---------------------------------
    // Corresponding to `val` property `UNIX_LINE_SEPARATOR` in Kotlin
    private static final String UNIX_LINE_SEPARATOR = "\n"; // backing field

    public static final String getUNIX_LINE_SEPARATOR() {   // getter
        return UNIX_LINE_SEPARATOR;
    }

    // ---------------------------------
    // Corresponding to `const val` property `CONST_UNIX_LINE_SEPARATOR` in Kotlin
    public static final String CONST_UNIX_LINE_SEPARATOR = "\n";    // backing field
}
```

### The perspective of Java on top-level `var` properties in Kotlin
In Java code, you can access top-level `var` properties through their accessor methods (i.e., getter and setter).  
For `var opCount = 0` in Kotlin code, the equivalence relation is as follows:
| Kotlin code               | Equivalent Java code                                                |
| -------------------------------------- | ----------------------------------------------------------- |
| Backing field                      | `private static int opCount = 0;`  |
| Getter       | `public static final int getOpCount()`    |
| Setter | `public static final void setOpCount(int value)`  |

- Note that you cannot access the property directly through the name `opCount`.
- When accessing from Java code, you can use just its getter `getOpCount` and setter `setOpCount`.

### The perspective of Java on top-level `val` properties in Kotlin
Similarly, you can access top-level `val` propertiies through ther accessor method (i.e., getter).
For `val UNIX_LINE_SEPARATOR = "\n"` in Kotlin code, the equivalence relation is as follows:
| Kotlin code               | Equivalent Java code                                                |
| -------------------------------------- | --- |
| Backing field                      | `private static final String UNIX_LINE_SEPARATOR = "\n";`  |
| Getter       | `public static final String getUNIX_LINE_SEPARATOR()`    |
| Setter | - |

- Note that the `final` keyword is added to the equivalent code of the backing field.
- Similar to the case of `var`, you cannot access the property directly through the name `UNIX_LINE_SEPARATOR`.
- Your way is just using its getter `getUNIX_LINE_SEPARATOR`.

### The perspective of Java on top-level `const val` properties in Kotlin
Unlike simple `val` which is just read-only property, there is a **compile-time constant**: `const val`.  
- Note that `const val` is available under special conditions:
  - only for primitive types and `String`
  - only at permitted positions: `object`, `companion object`, and top level
When you call a `const val` property from Java code, you can access it, directly using its name!  

| Kotlin code               | Equivalent Java code                                                |
| -------------------------------------- | --- |
| Backing field                      | `public static final String CONST_UNIX_LINE_SEPARATOR = "\n";`  |
| Getter       | -    |
| Setter | - |


In equivalent Java code, `const val` properties such as `CONST_UNIX_LINE_SEPARATOR` are exposed as `public static final`.
- Note that it is `public` unlike the case of simple `val` properties.

# Summary

```
(Kotlin) top-level function
→ (Java) static method in {FileName}Kt class

(Kotlin) top-level var
→ (Java) static field + getter + setter

(Kotlin) top-level val
→ (Java) static final field + getter

(Kotlin) top-level const val
→ (Java) public static final field
```
